TextCatcher-RegexTester 是一个基于Python的正则表达式文本提取工具，可帮助用户从冗长的文本中提取出想要的文本，匹配的模式可以通过选择文本和点击按键的方式自动创建，正则表达式可以简洁的生成。如果需要匹配的内容较为复杂，也可以手动修改正则表达式。

本软件由Xian Wu个人开发，为单一exe文件的GUI程序，可在windows系统上运行（如果要在其他操作系统上运行需要使用原代码重新打包），使用python 3.10和VS2022的环境，借助了ChatGPT、Gemini和Amazon Q的帮助。


如何使用：
1、通过open file加载需要提取的txt文件；
2、选中需要匹配的文字，点击set as match按钮；
3、选中已经设置为match的文字中需要提取的文字，点击set as capture；
4、选中已经设置为match的文字中不需要匹配的可能会变化的字符，点击set as not care；
5、通过上方的文字窗口观察和修改每段match字符对应的自动生成的regex，可以通过点击文字位置自动切换regex，也可以通过点击上下按键切换字符位置和对应的regex；
6、每次新建的regex都默认是序号最大的，可以通过Order Change：-1和+1的按键调整regex的序号；
7、新建的regex会有一个name，会显示在run之后的结果中作为每一列的标题，可以在regex name文本框中进行修改；
8、点击run按键，会根据已经设置好的regex自动搜索并匹配，最后结果会出现在右侧的文本框中。第一行默认是所有regex匹配的次数；
9、如果勾选了in sequence，不会每个regex单独匹配整个文本，而是会每个regex逐一匹配一次，然后继续匹配后方的文本；
10、如果勾选了row number，会将每个匹配的结果的行号也显示在单独的一列中。
11、通过set as match、set as capture、set as not care按钮编辑好的regex，可以通过save regex按键保存；
12、通过save regex按键保存好的文件，也可以通脱load regex按键加载并匹配其他的文件。需要注意，对新打开的文件，需要手动加载已保存的regex file，前一个文件已经编辑好的regex，如果打开了新文件不会自动保留下来。

注意：
1、set as match、set as capture、set as not care按钮会按照一定的规则转化文本，并不一定能完美的匹配用户的想法，可以转化后测试观察是否满足需求，如果不满足需要手动修改regex。
2、右上方有2个按钮，可以看到常用的regex和regex教程。


TextCatcher README – Last Edit: 2026.02.09 by Xian Wu

**TextCatcher-RegexTester** is a Python-based regex text extraction tool designed to help users extract specific data from lengthy documents. It simplifies the process by automatically generating concise regular expressions—users can create patterns simply by selecting text and clicking buttons. For more complex scenarios, the generated regex can also be manually refined.

Developed by **Xian Wu**, this software is a standalone GUI application (`.exe`) for Windows. (Note: Running it on other operating systems requires re-bundling the source code). It was built in a Python 3.10 and VS2022 environment, with technical assistance from ChatGPT, Gemini, and Amazon Q.

---

### How to Use

1. **Open File:** Click "Open File" to load the text file you want to process.
2. **Set as Match:** Select the text segment you want to match and click the **Set as Match** button.
3. **Set as Capture:** Within the segment already marked as "Match," select the specific text you wish to extract and click **Set as Capture**.
4. **Set as Not Care:** Select any characters within the "Match" segment that are variable or irrelevant, then click **Set as Not Care**.
5. **Edit Regex:** Use the upper text window to observe and modify the auto-generated regex for each segment. You can click on the text to switch between patterns or use the **Up/Down** buttons to navigate.
6. **Adjust Order:** New patterns are assigned the highest index by default. Use the **Order Change: -1/+1** buttons to reorder them.
7. **Name Patterns:** Each regex has a "Name" that serves as the column header in the results. You can edit this in the **Regex Name** textbox.
8. **Run:** Click **Run** to begin the search. Results will appear in the right-hand textbox, with the first row showing the total match count for each pattern.
9. **In Sequence:** If enabled, the tool will match patterns sequentially through the text rather than searching the entire document for each regex individually.
10. **Row Number:** Check this to display the line number for each match in a dedicated column.
11. **Save Regex:** Use the **Save Regex** button to export the patterns you've created.
12. **Load Regex:** Use **Load Regex** to apply saved patterns to different files. *Note: When opening a new text file, previously edited patterns are not automatically retained; you must reload the saved regex file manually.*

---

### Notes

* **Pattern Generation:** The "Set as..." buttons follow specific conversion rules and might not always perfectly capture your intent. Always test and manually refine the regex if the results don't meet your needs.
* **Resources:** Use the two buttons in the top-right corner to access **Common Regex Patterns** and a **Regex Tutorial**.
